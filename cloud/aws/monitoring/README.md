# AWS Monitoring

Monitoring services cover metrics, logs, traces, and configuration history.

### Decision Path

| Need | Start with |
| :-- | :-- |
| Collect metrics, logs, and alarms. | [CloudWatch](./cloudwatch.md) |

### When to Use AWS Monitoring

- You need a single place to see resource health and application metrics.
- You need log aggregation and alerting tied to AWS resources.
- You want managed integration with other AWS services.
