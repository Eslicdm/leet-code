# AWS Serverless

Serverless services cover orchestration and real-time APIs without managing servers.

### Decision Path

| Need | Start with |
| :-- | :-- |
| Orchestrate workflows as state machines. | [Step Functions](./step-functions.md) |
| Build managed GraphQL APIs with real-time subscriptions. | [AppSync](./appsync.md) |

### When to Use AWS Serverless

- You need visual workflow orchestration with built-in error handling.
- You need a managed GraphQL endpoint with real-time subscriptions.
- You want to reduce glue code and operational overhead.
