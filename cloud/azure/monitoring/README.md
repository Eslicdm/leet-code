# Azure Monitoring

Monitoring services cover metrics, logs, traces, and alerting.

### Decision Path

| Need | Start with |
| :-- | :-- |
| Collect metrics, logs, and alerts across Azure resources. | [Azure Monitor](./azure-monitor.md) |

### When to Use Azure Monitoring

- You want a single pane of glass for Azure resources and applications.
- You need log analytics workspaces and KQL-based queries.
- You want managed integration with Entra ID, Action Groups, and alerts.
